The problem

Our app had been a testing ground for features, leading to bloat and a messy user experience.

Solution

Prune away what is not used, enhance the experience for the flows that are loved.

This was an experiment on the side of our regular product (a what's on tv app) which demanded that we would iterate quickly. The idea was to build a native app at some point but to start, our 2 man team would use the latest technology ( as of 2013) to build a "web app".

The starting point was "what's new". Which would in time turn into a personalized flow based on what the user was saving to their list. So if you were heavy into sci-fi we would eventually highlight sci-fi in your feed. At the time this made a lot of sense because both netflix and HBO mostly had content that had already been aired on TV.

This browser does not support the video element.

When the user tried to save a movie to their watchlist, that action initiated a signup flow if the user was not already signed in. The idea was that since you had already found something to save, signing up would be an easy decision.
